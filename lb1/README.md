# Симуляція інтернет-замовлень (Java, багатопоточність)

Невелика симуляція інтернет-магазину, яка демонструє роботу з багатопоточністю та синхронізацією в Java:

* **Адміністратор** (Admin) поповнює запаси товарів.
* **Покупці** (Buyers) намагаються придбати товари.
* **Семафор** для кожного товару контролює залишок.
* **Робочі години** (WorkingHours) керують відкриттям та закриттям магазину.
* Акуратне завершення роботи програми з логами в реальному часі.

## Як працює програма

1. **`WorkingHours`** періодично відкриває та закриває магазин (цикл OPEN → CLOSE).
2. **`Admin`** з певним інтервалом додає випадкову кількість товару до випадкового продукту.
3. Кожен **`Buyer`**:

   * якщо магазин закритий — чекає та виводить повідомлення `WAITING (store closed)`;
   * якщо магазин відкритий — обирає бажаний або випадковий товар;
   * намагається придбати **1 одиницю** (`Semaphore.tryAcquire()`):

     * якщо вдалося — `BOUGHT ...`;
     * якщо ні — `SOLD OUT ...`.
4. Через ~5 секунд головний потік зупиняє всі інші та виводить фінальні залишки товарів.

>  Основний механізм — **`Semaphore`**: дозвіл = одиниця товару. `acquire` → купівля, `release` → поповнення.

## Структура проєкту

```
src/
└── lb1/
    └── Main.java      (всі класи в одному файлі: Product, Store, Admin, Buyer, WorkingHours)
```

## Вимоги

* JDK 11+ (також працює на JDK 8).

## Як запустити

Перейти у папку **на один рівень вище** `lb1` (бо в коді використано `package lb1;`):

```bash
# компіляція
javac lb1/Main.java

# запуск
java -cp . lb1.Main
```

## Завершення роботи

Головний потік надсилає всім потокам сигнал `interrupt`, а ті коректно зупиняються після переривання.
